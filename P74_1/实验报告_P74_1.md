# 数据结构实验报告 —— 顺序表

## 一、实验名称

**顺序表（SqList）的实现与应用**

## 二、实验目的

1. 掌握顺序存储结构的基本概念和特点。
2. 理解动态数组的扩容与缩容机制。
3. 实现顺序表的增、删、改、查等基本操作，并通过测试程序验证功能的正确性。

## 三、实验内容与过程

### 3.1 数据结构设计

本实验实现了一个类模板 `Sqlist<E>`，底层使用**动态数组**存储数据元素。顺序表将数据元素连续存放在一段内存空间中，逻辑上相邻的元素在物理存储位置上也相邻，具有**随机存取**特性（通过下标可在 O(1) 时间内访问任意元素）。

#### 核心成员变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `data` | `E*` | 指向堆上分配的动态数组 |
| `size` | `int` | 当前表中元素的个数 |
| `capacity` | `int` | 数组当前的最大容量 |
| `INIT_CAPACITY` | `static const int` | 初始容量，值为 10 |

#### 构造函数与析构函数

- **默认构造函数**：分配容量为 10 的数组，初始元素个数为 0。
- **带参构造函数**：允许用户指定初始容量。
- **析构函数**：释放动态数组内存，防止内存泄漏。
- 遵循**三法则（Rule of Three）** 的基本要求。

### 3.2 核心算法实现

#### 3.2.1 扩容与缩容策略 —— `resize`

```cpp
void resize(int newCapacity)
```

当数组已满（`size == capacity`）时，调用 `resize(2 * capacity)` 将容量**翻倍**；当元素个数少于容量的 1/4 且容量大于初始容量时，调用 `resize(capacity / 2)` 将容量**减半**。

扩容/缩容的具体步骤：
1. 在堆上申请新的数组 `temp`，大小为 `newCapacity`。
2. 将原数组中的 `size` 个元素复制到新数组。
3. 释放原数组内存，将 `data` 指针指向新数组，更新 `capacity`。

采用**二倍扩容**策略确保了 `addLast`（末尾插入）操作的**均摊时间复杂度为 O(1)**。

#### 3.2.2 插入操作 —— `insert(int index, E e)`

在第 `index` 位置（1-based）插入元素 `e`：
1. 调用 `checkIndex(index)` 检查插入位置是否合法（`1 ≤ index ≤ size+1`）。
2. 若数组已满，先扩容。
3. 将 `index` 及之后的所有元素**向后移动一个位置**（从后往前遍历）。
4. 将元素 `e` 放入 `data[index-1]`，`size` 加 1。

时间复杂度：**O(n)**（因为需要移动元素）。

#### 3.2.3 删除操作 —— `remove(int index)`

删除第 `index` 位置（1-based）的元素：
1. 调用 `checkIndexPosition(index)` 检查访问位置是否合法（`1 ≤ index ≤ size`）。
2. 保存被删除元素的值。
3. 将 `index` 之后的所有元素**向前移动一个位置**（从前往后遍历）。
4. 将最后一个位置置为默认值，`size` 减 1。
5. 检查是否需要缩容。

时间复杂度：**O(n)**。

#### 3.2.4 查找操作

- **`getE(int index)`**：按位置查找，直接通过下标访问 `data[index-1]`，时间复杂度 **O(1)**。
- **`getIndex(E e)`**：按值查找，遍历数组返回匹配的所有位置，时间复杂度 **O(n)**。

#### 3.2.5 边界检查

- `checkIndex(int index)`：用于插入时的位置检查，允许 index = size+1（在末尾插入）。
- `checkIndexPosition(int index)`：用于访问、删除、修改时的位置检查，不允许超出 size。

两者均在不合法时抛出 `std::out_of_range` 异常，保证了程序的健壮性。

### 3.3 测试程序（exp2-1.cpp）

测试程序创建了一个 `Sqlist<char>` 类型的顺序表，依次执行以下操作：

1. `addLast` 依次添加元素 'a'、'b'、'c'、'd'、'e'，打印顺序表。
2. 调用 `isEmpty()` 检查是否为空，输出 `0`（false）。
3. 调用 `getE(3)` 获取下标 3 的元素（即 'c'），输出 `c`。
4. 调用 `getIndex('a')` 查找值为 'a' 的位置，输出 `1`。
5. 调用 `insert(4, 'f')` 在位置 4 插入 'f'，打印结果：`a b c f d e`。
6. 调用 `remove(3)` 删除位置 3 的元素（即 'c'），打印最终结果：`a b f d e`。

#### 运行输出

```
a b c d e
0
c
1
a b c f d e
a b f d e
```

## 四、实验总结

通过本次实验，我深入理解了顺序表（线性表的顺序存储结构）的实现原理。顺序表利用连续内存空间的优势实现了 O(1) 的随机访问，但插入和删除操作需要移动大量元素，时间复杂度为 O(n)。动态扩容与缩容机制的引入使顺序表能够动态适应数据规模的变化，在保证高效存储的同时避免了固定容量带来的浪费或溢出问题。编写过程中对异常处理（边界检查）的实践也增强了程序的健壮性。

## 五、心得体会

通过亲手实现顺序表的模板类，我对 C++ 模板编程、动态内存管理和均摊分析有了更直观的认识。特别是二倍扩容策略的时间复杂度分析让我意识到，虽然单次扩容开销较大，但从整体操作序列来看，平均代价是常数级别的。实验中体会最深的是边界条件的处理——插入和删除时索引的合法性判断必须严谨，否则极易引发数组越界等运行时错误。此外，将数据结构封装为类模板的方式使代码复用了更高，为后续其他数据结构的实现打下了良好基础。
