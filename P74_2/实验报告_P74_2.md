# 数据结构实验报告 —— 单链表

## 一、实验名称

**单链表（LinkList）的实现与应用**

## 二、实验目的

1. 掌握链式存储结构的基本概念和特点。
2. 理解单链表中头指针、尾指针、头结点的作用。
3. 实现单链表的头插、尾插、按位置插入、头删、尾删、按位置删除以及查找等基本操作。

## 三、实验内容与过程

### 3.1 数据结构设计

本实验实现了一个类模板 `Linklist<E>`，底层使用**带头结点的单链表**存储数据元素。与顺序表不同，链表中的元素在内存中是**非连续存储**的，每个结点（Node）包含数据域 `data` 和指向下一个结点的指针域 `next`，逻辑上相邻的元素通过指针链接。

#### 核心结构

```cpp
struct Node {
    E data;
    Node* next;
    Node(E value) : data(value), next(nullptr) {}
};
```

| 成员变量 | 类型 | 说明 |
|----------|------|------|
| `head` | `Node*` | 头结点指针（不存储数据，仅作为哨兵） |
| `tail` | `Node*` | 尾结点指针（方便 O(1) 尾插） |
| `size` | `int` | 链表中元素的个数 |

#### 设计要点

- **头结点（head）**：不存储实际数据，其 `next` 指针指向链表中第一个数据结点。头结点的存在统一了空链表和非空链表的插删操作逻辑，避免了对空链表特殊情况的单独判断。
- **尾指针（tail）**：始终指向链表的最后一个数据结点，使得尾插（addLast）操作只需 O(1) 时间。当链表为空时，`tail` 指向 `head`。
- 采用 **1-based 索引**，与用户接口一致。

### 3.2 核心算法实现

#### 3.2.1 插入操作

##### 头插 —— `addFirst(E e)`

1. 创建新结点 `newNode`。
2. 将 `newNode->next` 指向 `head->next`（原第一个数据结点）。
3. 将 `head->next` 指向 `newNode`。
4. 若链表原本为空（`size == 0`），更新 `tail = newNode`。
5. `size` 加 1。

时间复杂度：**O(1)**。

##### 尾插 —— `addLast(E e)`

利用尾指针直接操作：

1. 创建新结点 `newNode`。
2. 将 `tail->next` 设为 `newNode`。
3. 更新 `tail = newNode`。
4. `size` 加 1。

时间复杂度：**O(1)**（因为有尾指针）。

##### 按位置插入 —— `insert(int index, E e)`

在第 `index` 位置插入元素 `e`：

1. 检查 `index` 是否合法（`1 ≤ index ≤ size+1`）。
2. 从 `head` 出发，将指针 `current` 移动到第 `index-1` 个结点。
3. 将 `newNode->next` 指向 `current->next`。
4. 将 `current->next` 设为 `newNode`。
5. 若新结点插入在尾部，更新 `tail`。

时间复杂度：**O(n)**（需要遍历定位）。

#### 3.2.2 删除操作

##### 头删 —— `removeFirst()`

1. 检查链表是否为空。
2. 保存 `head->next`（第一个数据结点）。
3. 将 `head->next` 指向第二个数据结点。
4. 删除原第一个数据结点，释放内存。
5. 若删除后链表为空，将 `tail` 重置为 `head`。

时间复杂度：**O(1)**。

##### 尾删 —— `removeLast()`

由于单链表无法直接从尾结点回溯到前驱结点，必须从头开始遍历：

1. 从 `head` 出发，找到 `tail` 的前驱结点 `current`（即 `current->next == tail`）。
2. 删除 `tail` 结点。
3. 将 `current->next` 设为 `nullptr`。
4. 更新 `tail = current`。

时间复杂度：**O(n)**（需要遍历找到倒数第二个结点）。这是单链表尾删操作的固有缺陷。

##### 按位置删除 —— `remove(int index)`

与 `insert` 类似，先遍历到第 `index-1` 个结点，然后执行常规的单链表删除操作。若删除的是尾结点，需同步更新 `tail`。

时间复杂度：**O(n)**。

#### 3.2.3 查找操作

- **`getE(int index)`**：从头结点后的第一个数据结点开始，依次移动指针到目标位置，返回结点数据。时间复杂度 **O(n)**（不支持随机存取）。
- **`set(int index, E e)`**：与 `getE` 流程相同，但修改目标结点的数据域并返回旧值。

### 3.3 测试程序（exp2-2.cpp）

测试程序创建了一个 `Linklist<char>`，依次执行以下操作：

1. `addLast` 依次添加字符 'a'、'b'、'c'、'd'、'e'，调用 `printLinklist()` 输出。
2. 调用 `getSize()` 输出元素个数 `5`。
3. 调用 `isEmpty()` 检查是否为空，输出 `0`（false）。
4. 调用 `getE(3)` 获取第 3 个元素，输出 `c`。
5. 调用 `insert(4, 'f')` 在第 4 位插入 'f'，输出 `a b c f d e`。
6. 调用 `remove(3)` 删除第 3 个元素，输出 `a b f d e`。

#### 运行输出

```
a b c d e
5
0
c
a b c f d e
a b f d e
```

## 四、实验总结

单链表相比于顺序表，插入和删除操作不需要移动大量元素，只需修改指针指向；但在查找和按位置访问时，必须从头结点顺序遍历，无法像顺序表那样 O(1) 随机访问。通过头结点和尾指针的设计，头插和尾插的效率达到了 O(1)。本实验中尾删操作的 O(n) 复杂度是单链表的固有特性，若需要高效的尾删，可采用双向链表结构。

## 五、心得体会

在实现单链表的过程中，我深刻体会到了指针操作的严谨性要求。插入和删除时指针的修改顺序必须谨慎——稍有不慎就会造成链表断裂或内存泄漏。头结点的引入大大简化了代码逻辑，特别是处理空链表和头插/头删时不再需要特殊判断。同时，`getHeadNext()` 接口在 P77_12 题目中的实际应用也让我认识到，适当暴露内部结构可以支持更灵活的外部算法扩展。
