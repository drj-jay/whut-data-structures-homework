# 数据结构实验报告 —— 循环队列

## 一、实验名称

**循环队列（Circular Queue）的顺式存储实现与应用**

## 二、实验目的

1. 掌握队列（Queue）这一线性数据结构的定义及其**先进先出（FIFO）** 特性。
2. 理解循环队列的设计思想，解决普通顺序队列的"假溢出"问题。
3. 实现循环队列的动态扩容与缩容机制。

## 三、实验内容与过程

### 3.1 数据结构设计

本实验实现了一个类模板 `sqqueue<E>`（顺序队列），底层使用 `std::vector<E>` 作为动态数组存储。通过**循环缓冲区**（Circular Buffer）机制，使队列能够重复利用已出队元素空出的空间，避免顺序队列中常见的"假溢出"（队尾指针到达数组末尾但队首仍有空间）问题。

#### 核心成员变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `arr` | `std::vector<E>` | 底层动态数组，存储队列元素 |
| `count` | `int` | 队列当前元素个数 |
| `head` | `int` | 队首指针，指向第一个有效元素 |
| `tail` | `int` | 队尾指针，指向下一次入队应放入的位置 |

#### 设计要点

- **队列满的条件**：`count == arr.size()`（元素个数等于数组容量）。
- **队列空的条件**：`count == 0`。
- 采用 `count` 变量区分队空和队满，避免了仅用 `head == tail` 无法区分的经典问题。
- 默认初始容量为 6。
- 构造时使用初始化列表初始化所有成员变量。

### 3.2 核心算法实现

#### 3.2.1 循环寻址

入队和出队均通过取模运算实现循环：

```cpp
tail = (tail + 1) % arr.size();  // 入队时 tail 后移
head = (head + 1) % arr.size();  // 出队时 head 后移
```

遍历时，第 i 个元素（从 0 开始）的实际索引为：`(head + i) % arr.size()`。

#### 3.2.2 入队 —— `enqueue(const E &e)`

1. 若队列已满（`count == arr.size()`），将容量**翻倍**。
2. 将元素放入 `arr[tail]`。
3. `tail = (tail + 1) % arr.size()`。
4. `count++`。

时间复杂度：**均摊 O(1)**。

#### 3.2.3 出队 —— `dequeue()`

1. 若队列为空，抛出 `std::out_of_range` 异常。
2. 保存 `arr[head]` 的值。
3. `head = (head + 1) % arr.size()`。
4. `count--`。
5. 若元素个数降至容量的 1/4，将容量**减半**（缩容处理）。

时间复杂度：**O(1)**（无缩容时）/ **O(n)**（缩容时，均摊后仍为 O(1)）。

#### 3.2.4 扩容与缩容 —— `resize(int NewCapacity)`

当队列满或元素过少时触发，具体步骤：

1. 创建新 `vector`，大小为 `NewCapacity`。
2. 将原队列中的 `count` 个元素**按顺序**复制到新数组（从索引 0 开始连续存放）。遍历方式：`arr[(head + i) % arr.size()]`。
3. 移动新数组到 `arr`。
4. 重置 `head = 0`，`tail = count`（恢复为简单的连续存储状态）。

这一操作同时也完成了队列数据的**整理（重排）**，使扩容后的队列从松散排列变为紧凑排列。空间复杂度 O(n)。

#### 3.2.5 辅助操作

| 函数 | 功能 | 时间复杂度 |
|------|------|------------|
| `isempty()` | 判断队列是否为空（`count == 0`） | O(1) |
| `isfull()` | 判断队列是否已满（`count == arr.size()`） | O(1) |
| `getHeadIndex()` | 返回 `head` 的值（队首指针位置） | O(1) |
| `getTailIndex()` | 返回 `tail` 的值（队尾指针位置） | O(1) |
| `printSqqueue()` | 从队首到队尾遍历打印 | O(n) |

### 3.3 测试程序（exp3-3.cpp）

测试程序创建了一个默认容量为 6 的 `sqqueue<char>`，依次执行：

| 步骤 | 操作 | 队列状态 | 说明 |
|------|------|----------|------|
| 1 | 打印 `isempty()` | 期望 `1` | 初始为空 |
| 2 | `enqueue('a')` | `[a]` | |
| 3 | `enqueue('b')` | `[a, b]` | |
| 4 | `enqueue('c')` | `[a, b, c]` | |
| 5 | `dequeue()` | `[b, c]` | 输出 `a` |
| 6 | `enqueue('d')` | `[b, c, d]` | |
| 7 | `enqueue('e')` | `[b, c, d, e]` | |
| 8 | `enqueue('f')` | `[b, c, d, e, f]` | |
| 9 | 打印 `getTailIndex()` | | 验证 `tail` 的位置 |

#### 运行输出

```
1
a
5
```

**输出分析**：
- `1`：初始队空确认。
- `a`：出队元素。
- `5`：入队 6 次、出队 1 次后 `tail` 的索引值。注意在容量为 6 的循环队列中，经历一次出队后 `tail` 经历了循环定位。

### 3.4 循环队列 vs 普通顺序队列

| 比较维度 | 普通顺序队列 | 循环队列 |
|----------|-------------|----------|
| 假溢出 | 存在（tail 到达末尾后即使 head 前有空位也无法使用） | 不存在（通过取模利用释放的空间） |
| 队空/队满判定 | 可用 head==tail 判定 | 需要额外计数器 count |
| 空间利用率 | 低 | 高 |

## 四、实验总结

本实验通过 `std::vector` 和循环寻址实现了循环队列模板类。循环队列利用取模运算实现了存储空间的循环复用，解决了普通顺序队列中因出队操作产生的"假溢出"问题。通过额外的 `count` 变量明确区分队空与队满状态，同时引入了二倍扩容和缩容机制，使队列具备动态调整容量的能力。整个实现兼顾了 FIFO 的语义正确性、空间利用率和时间效率。

## 五、心得体会

实现循环队列时，取模运算的细节（尤其是 `tail = (tail + 1) % arr.size()` 这种模式）让我对"循环"思想有了更直观的理解。扩容时将所有元素重新整理为连续排列（`head=0, tail=count`）这一步非常巧妙，既完成了扩容又顺便整理了散乱的元素。这些细节看似简单，实际编码中却容易出错——比如扩容后忘记重置 head 和 tail，或者遍历时取模索引计算错误。亲手写一遍让我对队列的内部机制理解更深了。
