# 数据结构实验报告 —— 栈（基于 list 实现）

## 一、实验名称

**栈（Stack）的链式存储实现与应用**

## 二、实验目的

1. 掌握栈（Stack）这一线性数据结构的定义及其**后进先出（LIFO）** 特性。
2. 理解栈的链式存储实现方式。
3. 通过 C++ STL 容器 `std::list` 封装自定义栈类，实现入栈、出栈、判空等基本操作。

## 三、实验内容与过程

### 3.1 数据结构设计

本实验实现了一个类模板 `MyLinkedStack<E>`，底层使用 `std::list<E>` 容器作为存储结构。栈是一种操作受限的线性表，只允许在**栈顶**（同一端）进行插入和删除操作。

| 成员 | 类型 | 说明 |
|------|------|------|
| `list` | `std::list<E>` | 底层双向链表容器，真实存储数据 |

`std::list` 是 C++ STL 提供的**双向链表**，支持首尾 O(1) 的插入和删除。栈只需在**同一端**操作，因此本实现约定**以 list 尾部作为栈顶**：

- 入栈 (`push`) → 调用 `list.push_back()`，在链表尾部添加元素。
- 出栈 (`pop`) → 调用 `list.back()` 获取尾元素，再调用 `list.pop_back()` 删除。

### 3.2 核心操作实现

#### 3.2.1 入栈 —— `push(const E &e)`

```
void push(const E &e) { list.push_back(e); }
```

直接将元素追加到 list 尾部。时间复杂度 **O(1)**。

#### 3.2.2 出栈 —— `pop()`

```cpp
E pop() {
    if (list.empty()) {
        throw std::runtime_error("Stack is empty!");
    }
    E val = list.back();
    list.pop_back();
    return val;
}
```

执行步骤：
1. 判空：若栈为空，抛出 `std::runtime_error` 异常，防止非法出栈。
2. 获取栈顶元素 `list.back()`。
3. 删除栈顶元素 `list.pop_back()`。
4. 返回被删除的值。

时间复杂度 **O(1)**。

#### 3.2.3 判空 —— `isEmpty()`

```
bool isEmpty() { return list.empty(); }
```

直接调用 list 的 `empty()` 方法。时间复杂度 **O(1)**。

#### 3.2.4 获取栈顶位置编号 —— `getPopIndex()`

```
int getPopIndex() { return list.size(); }
```

返回当前栈中元素个数，即栈顶元素在 1-based 编号中的位置。可将此值理解为下次出栈元素的序号。时间复杂度 **O(1)**。

#### 3.2.5 遍历打印 —— `printStack()`

```
void printStack() {
    for (E i : list) { std::cout << i << " "; }
}
```

使用范围 for 循环遍历 list，从栈底到栈顶顺序输出。

### 3.3 算法正确性分析

**LIFO 特性的验证**：由于入栈固定使用 `push_back`（尾部追加），出栈固定使用 `pop_back`（尾部弹出），后入栈的元素总是先被弹出，严格遵循后进先出原则。

**异常安全性**：在 `pop()` 中加入了空栈检查，防止对空栈执行出栈操作导致的未定义行为。

### 3.4 测试程序（exp3-2.cpp）

测试程序创建了一个 `MyLinkedStack<char>` 对象，依次执行：

1. 打印 `isEmpty()` → 期望输出 `1`（true，栈为空）。
2. 依次 `push` 字符 'a'、'b'、'c'、'd'、'e'。
3. 再次打印 `isEmpty()` → 期望输出 `0`（false，栈非空）。
4. 打印 `getPopIndex()` → 期望输出 `5`（栈中有 5 个元素）。
5. 最后一次打印 `isEmpty()` → 期望输出 `0`。

#### 运行输出

```
1
0
5
0
```

## 四、实验总结

本次实验通过封装 STL 的 `std::list` 容器实现了链式栈。`std::list` 作为双向链表，在首尾进行插入和删除的时间复杂度均为 O(1)，非常适合作为栈的底层结构。通过限制操作接口（只暴露 `push` 和 `pop`），将通用的线性表约束为具有 LIFO 特性的栈，体现了"操作受限的线性表"这一核心概念。

## 五、心得体会

虽然在 C++ 中直接使用 `std::stack` 也很方便，但亲手基于 `std::list` 封装一遍栈，使我对栈的**抽象数据类型（ADT）** 本质有了更清晰的认识：栈的核心不是底层用什么结构存储，而是**接口的约束**——只允许在一端进出。这种"封装+限制接口"的思想在软件工程中非常普遍，也是数据结构课程的要义之一。
