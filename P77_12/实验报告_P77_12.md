# 数据结构实验报告 —— 单链表的综合应用

## 一、实验名称

**单链表的综合应用 —— 大整数加法与快慢指针**

## 二、实验目的

1. 在单链表基础上实现链表反转（`reverse`）与中间结点查找（`getMiddle`，快慢指针法）。
2. 综合运用链表操作解决实际问题：利用链表存储大整数，并通过单链表实现两个任意长度大整数的加法运算。
3. 理解数据结构在实际问题中的灵活运用。

## 三、实验内容与过程

### 3.1 数据结构设计

本实验复用了 P74_2 中已经实现的 `Linklist<E>` 模板类，并在其基础上新增了以下功能。

#### 新增成员函数

| 函数 | 功能 | 时间复杂度 |
|------|------|------------|
| `reverse()` | 原地反转链表 | O(n) |
| `getMiddle()` | 返回中间结点的值（快慢指针） | O(n) |
| `getHeadNext()` | 获取第一个数据结点的指针（供外部算法使用） | O(1) |

#### 新增友元函数

| 函数 | 功能 |
|------|------|
| `addLinklist(L1, L2, result)` | 计算两个用链表表示的大整数之和，结果存入 `result` |

### 3.2 核心算法实现

#### 3.2.1 链表反转 —— `reverse()`

采用**三指针迭代法**原地反转单链表：

1. 若链表元素个数 ≤ 1，直接返回（无需反转）。
2. 初始化三个指针：
   - `prev = nullptr`（已反转部分的头指针）
   - `current = head->next`（当前处理的结点）
3. 遍历链表，对每个结点执行：
   - 用 `next` 暂存 `current->next`（保存后继）。
   - 将 `current->next` 改为指向 `prev`（反转链接方向）。
   - `prev` 和 `current` 同步后移。
4. 遍历结束后：
   - 原尾结点（`head->next`）成为新尾结点，更新 `tail = head->next`。
   - `prev` 成为新头结点，更新 `head->next = prev`。

```cpp
void reverse() {
    if (size <= 1) return;
    Node* prev = nullptr;
    Node* current = head->next;
    while (current != nullptr) {
        Node* next = current->next;
        current->next = prev;
        prev = current;
        current = next;
    }
    tail = head->next;
    head->next = prev;
}
```

该算法只需一次遍历，空间复杂度为 O(1)。

#### 3.2.2 快慢指针找中间结点 —— `getMiddle()`

利用**快慢指针（Floyd 判圈算法的变体）** 在单次遍历中找到中间结点：

1. 初始化 `slow` 和 `fast` 指针均指向 `head->next`。
2. 循环条件：`fast->next != nullptr && fast->next->next != nullptr`。
3. 每次循环：`slow` 前进一步，`fast` 前进两步。
4. 当 `fast` 无法继续前进时，`slow` 所指即为中间结点。

**中间位置规则**：
- 奇数个结点（如 5 个）：`slow` 恰好停在正中间（第 3 个）。
- 偶数个结点（如 4 个）：`slow` 停在下中位数位置（第 2 个），符合题目要求的"向下取整"规则。

时间复杂度 O(n)，空间复杂度 O(1)。

#### 3.2.3 大整数加法 —— `addLinklist`

本题的核心应用：将两个用**单链表**表示的任意长度十进制大整数相加。

##### 输入格式

用户输入两个由纯数字字符组成的字符串，通过 `stringToLinklist` 函数逐字符转换为链表结点。例如输入 `"123"` 转换为链表：`[1] → [2] → [3]`。

##### 加法算法步骤

考虑到人类习惯的高位在前（如 `123` 链表为 `1→2→3`），但加法需从低位开始计算，因此算法采用"反转—相加—头插"策略：

1. **反转链表**：分别调用 `L1.reverse()` 和 `L2.reverse()`，使低位数在链表头部。
2. **逐位相加**：
   - 初始化进位 `carry = 0`。
   - 同时遍历两个链表，每次取出当前位的数字（指针为空的位视为 0）。
   - 计算 `sum = carry + x + y`，新进位 `carry = sum / 10`，当前位 `= sum % 10`。
   - 使用 `addFirst`（头插法）将当前位插入 `result` 链表。
   - 头插法使结果从低位到高位自然恢复为正序。
3. **处理最终进位**：若遍历结束后 `carry != 0`，则头插一个值为 `carry` 的结点。
4. **恢复原链表**：再次反转 L1 和 L2，使其恢复到原始状态。

##### 关键代码

```cpp
void addLinklist(Linklist<int>& L1, Linklist<int>& L2, Linklist<int>& result) {
    L1.reverse();
    L2.reverse();
    int carry = 0;
    auto p1 = L1.getHeadNext();
    auto p2 = L2.getHeadNext();
    while (p1 != nullptr || p2 != nullptr || carry != 0) {
        int x = p1 != nullptr ? p1->data : 0;
        int y = p2 != nullptr ? p2->data : 0;
        int sum = carry + x + y;
        carry = sum / 10;
        result.addFirst(sum % 10);
        if (p1 != nullptr) p1 = p1->next;
        if (p2 != nullptr) p2 = p2->next;
    }
    L1.reverse();
    L2.reverse();
}
```

时间复杂度 O(max(m, n))，其中 m、n 分别为两个输入数字的位数。

### 3.3 测试程序（exp2-12.cpp）

测试程序实现了完整的大整数加法流程：

1. 提示用户输入两个数字字符串。
2. 调用 `stringToLinklist` 将字符串转换为 `Linklist<int>`。
3. 调用 `addLinklist` 执行加法，结果存储在 `result` 链表中。
4. 打印 `result` 链表。
5. 调用 `result.getMiddle()` 输出结果数字的中间位数。

#### 运行示例

```
write the first number string: 123
write the second number string: 4567
result: 4690
middle of result: 6
```

#### 输入异常处理

`stringToLinklist` 函数使用 `std::isdigit()` 对每个字符进行校验，若包含非数字字符则抛出 `std::invalid_argument` 异常，确保输入合法性。

## 四、实验总结

本次实验在单链表基础上进行了两个重要扩展：链表反转（三指针法）和快慢指针求中间结点，两者均体现了使用少量指针巧解链表问题的思想。随后，综合运用反转、头插、友元函数等操作，实现了大整数加法的链表解决方案，体现了数据结构在解决实际问题中的核心作用。整个过程中，链表的灵活性得到了充分发挥——反转改变了遍历方向，头插法巧妙地将加法结果恢复为正序，符合工程常见的"空间换时间"或"结构调整"思路。

## 五、心得体会

本次实验中，我最大的收获是深入理解了链表指针操作的巧妙之处。三指针反转算法看似简单，但其中指针的修改顺序必须精准无误，否则会丢失结点造成链表断裂。快慢指针法则是算法设计中"双指针"思想的典型体现，通过不同步速的指针在一趟遍历中获取结构性信息。大整数加法的实现让我意识到，数据结构不是孤立存在的——选择合适的数据结构（链表 vs 数组）和操作策略（反转、头插）才能高效解决问题。当数字长度远超出内置整数类型的表示范围时，链表提供了一种优雅的解决方案，这正是数据结构课程的核心价值所在。
